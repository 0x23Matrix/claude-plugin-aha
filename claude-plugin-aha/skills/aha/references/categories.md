# Insight Categories

Use these categories to classify aha moments. One insight may have a primary category and optional secondary tags.

## Categories

| Category | Slug | Description |
|----------|------|-------------|
| Programming Patterns | `pattern` | Design patterns, idioms, elegant solutions, coding techniques |
| Architecture | `architecture` | System design, component relationships, scalability approaches |
| Debugging | `debugging` | Root cause analysis techniques, diagnostic approaches, common pitfalls |
| Tool & Workflow | `tool` | CLI tools, IDE features, automation tricks, dev workflow improvements |
| Performance | `performance` | Optimization techniques, profiling insights, resource management |
| Data & Algorithms | `algorithm` | Data structures, algorithmic thinking, complexity insights |
| Product & UX | `product` | User experience insights, product thinking, feature design |
| Domain Knowledge | `domain` | Industry-specific or business logic insights |
| Mental Model | `mental-model` | Conceptual breakthroughs, ways of thinking, analogies |
| Communication | `communication` | Technical writing, documentation, explaining complex ideas |

## Tagging Guidelines

- Each insight gets exactly **one primary category** (the most fitting)
- Add 1-3 **free-form tags** for cross-referencing (e.g., `#typescript`, `#react`, `#concurrency`)
- Tags should be lowercase, hyphenated, reusable across entries
